# Yapsolutely credential setup checklist

Use this while the remaining implementation work continues.

This file tells you:

- which credentials are actually needed
- where to get them
- which ones are free vs paid
- the rough cost expectations for MVP testing
- exactly which `.env` keys they map to

> Pricing changes often. Treat all dollar amounts below as **rough planning numbers**, not a promise. Verify current pricing on each provider’s pricing page before purchasing.

---

## 1) What you need to collect

### Required for real end-to-end MVP validation

These are the ones we need before we can truthfully say the live demo flow works end to end.

1. **Supabase project credentials**
2. **Anthropic API key**
3. **Deepgram API key**
4. **Twilio account + phone number**
5. **Web deployment URL**
6. **Voice runtime deployment URL**

### Generated by you locally for security

These are not bought from vendors. You create them yourself.

1. `AUTH_SECRET`
2. `RUNTIME_SHARED_SECRET`

### Optional / depends on your setup

1. `TRANSFER_NUMBER` — only if you want escalation/transfer behavior
2. `TWILIO_SMS_FROM_NUMBER` — can usually be the same as your Twilio phone number for MVP

## 2) Best order to get everything

Follow this order so nothing blocks the next step:

1. Create **Supabase** project
2. Create **Anthropic** account and API key
3. Create **Deepgram** account and API key
4. Create **Twilio** account, buy a phone number, gather webhook/runtime values
5. Create **Vercel** project for `apps/web`
6. Create **Railway** project for `apps/voice`
7. Fill `.env` / production envs with the collected values

## 3) Step-by-step by provider

## Supabase

### What it is used for

- Postgres database for Prisma
- stores users, agents, numbers, calls, transcripts, events

### Usually free or paid?

- **Usually free to start** on the free tier
- For MVP dev/test, free tier is typically enough
- Paid plan is only needed later if you outgrow the free limits or need production guarantees

### Rough cost

- **Start:** $0 likely fine
- **Later production:** usually a monthly paid plan if you need stronger limits / uptime

### How to get it

1. Create a Supabase account
2. Create a new project
3. Choose a region close to where you’ll test from
4. Wait for the database to finish provisioning
5. In project settings / database connection area, copy:
	- Postgres connection string
	- direct connection string if available
6. In API settings, copy:
	- project URL
	- anon key
	- service role key

### Env vars to fill

- `DATABASE_URL`
- `DIRECT_URL`
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

### What to send me when ready

- database connection string
- direct connection string
- Supabase project URL
- anon key
- service role key

---

## Anthropic

### What it is used for

- live LLM responses in the voice runtime
- tool-calling logic during calls

### Usually free or paid?

- **Usually paid / usage-based**
- Some accounts may have limited promotional credit, but do **not** plan around that

### Rough cost

- Budget at least **$5–$20+** to be safe for development/testing
- Actual spend depends on call volume and model usage

### How to get it

1. Create an Anthropic account
2. Add billing / payment method if required
3. Go to API keys
4. Create a new key
5. Copy it immediately and store it safely

### Env vars to fill

- `ANTHROPIC_API_KEY`
- optional model settings already defaulted in repo:
  - `ANTHROPIC_MODEL`
  - `ANTHROPIC_MAX_TOKENS`

### What to send me when ready

- `ANTHROPIC_API_KEY`

## Twilio

### What it is used for

- real phone number
- inbound calls
- voice webhooks
- media streams
- SMS tool actions

### Usually free or paid?

- **Paid for real usage**
- Trial accounts exist, but for a clean inbound-call MVP you should expect to pay

### Rough cost

For MVP testing, plan roughly:

- phone number: **about $1–$2+ / month** for a basic local number
- voice usage: **per-minute charges**
- SMS: **per-message charges**
- safe starter budget: **$10–$30** is usually enough for early testing

> Exact cost depends on country, number type, call destination, SMS destination, and current Twilio pricing.

### How to get it

1. Create a Twilio account
2. Verify your account as required
3. Add billing / top-up / payment method
4. Buy one phone number with:
	- **Voice** capability
	- **SMS** capability if you want SMS confirmation tool testing
5. Copy from the Twilio console:
	- Account SID
	- Auth Token
	- purchased phone number
6. If you buy a separate SMS-capable number, copy that too

### Env vars to fill

- `TWILIO_ACCOUNT_SID`
- `TWILIO_AUTH_TOKEN`
- `TWILIO_PHONE_NUMBER`
- `TWILIO_SMS_FROM_NUMBER`
- optional: `TRANSFER_NUMBER`

### What to send me when ready

- `TWILIO_ACCOUNT_SID`
- `TWILIO_AUTH_TOKEN`
- purchased Twilio number
- SMS from number if different

## 4) Secrets you generate yourself

## AUTH_SECRET

### What it is used for

- session/auth signing

### Paid?

- **No**

### How to create it

Use any secure random string generator.

Recommended characteristics:

- at least 32 bytes of randomness
- long random string
- do not reuse from another project

### Env var

- `AUTH_SECRET`

---

## RUNTIME_SHARED_SECRET

### What it is used for

- secures runtime-to-web internal API calls

### Paid?

- **No**

### How to create it

Generate another strong random string separate from `AUTH_SECRET`.

### Env var

- `RUNTIME_SHARED_SECRET`
